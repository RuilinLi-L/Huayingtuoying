# 华音拓影正式接入与素材替换指南

本文档面向项目组内部，用来说明这套一期 WebAR 系统的实际接入方式：  
如何替换正式海报、识别图、`.mind` 文件、3D 模型、分轨音频，以及如何把 NFC 标签和二维码接到系统里。

---

## 1. 先理解这套系统的工作方式

这不是一个“手机直接读 NFC 后在浏览器里解析数据”的方案，而是一个更容易落地的方案：

1. 把一个 HTTPS 链接写进 NFC 标签；
2. 或者把同一个链接做成二维码；
3. 用户打开链接后进入 H5；
4. H5 根据 `entryId` 加载对应条目的海报、识别图、AR 场景、模型、音频和知识卡。

也就是说：

- NFC 只是“入口”
- 二维码也是“入口”
- 真正的内容加载逻辑在前端项目里

当前深链格式是：

```text
/?entry=<entryId>
```

例如：

```text
/?entry=violin-dialogue
```

---

## 2. 代码里最重要的几个位置

### 2.1 内容总入口

所有条目内容都在：

`src/data/entries.json`

这里相当于系统的一份静态“内容数据库”。  
每新增一个乐器条目，本质上就是往这里新增一个对象。

相关类型定义在：

`src/types/manifest.ts`

其中最关键的是 `EntryManifest`、`AudioStem`、`KnowledgeCard`。

### 2.2 链接与路由

深链生成和条目查找在：

`src/lib/entries.ts`

这里负责：

- 根据 `entryId` 找条目
- 生成入口页路径
- 生成体验页路径
- 生成 `/?entry=<entryId>` 深链

页面路由在：

`src/App.tsx`

当前有三层：

- 首页 `/`
- 条目入口页 `/entry/:entryId`
- 体验页 `/experience/:entryId`

### 2.3 WebAR 场景

AR 场景组件在：

`src/components/ar/MindArScene.tsx`

这里主要做三件事：

- 加载 MindAR 和 A-Frame
- 把 `trackingTargetSrc` 作为识别目标
- 把 `modelUrl` 作为 3D 模型加载到识别图上方

### 2.4 音频分轨

音频引擎在：

`src/lib/audio/AudioEngine.ts`

这里支持：

- 多轨同步播放
- 单轨开关
- 独奏
- 总音量
- 为后续空间音频保留 `stereoPan`、`gain` 参数

### 2.5 NFC / 二维码导出

导出脚本在：

`scripts/generate-links.mjs`

导出结果在：

`generated/nfc-links.json`

和：

`generated/qrcodes/*.svg`

## 3. 每个正式条目需要准备什么

每个乐器条目，建议至少准备以下资源：

### 3.1 视觉资源

- `posterImage`：入口页和 fallback 用的海报图
- `targetImage`：给用户看的识别图
- `trackingTargetSrc`：MindAR 真正识别使用的 `.mind` 文件

### 3.2 3D 资源

- `modelUrl`：乐器、乐手、舞台或组合场景模型

建议格式：

- 优先 `glb`
- 也可 `gltf`

当前示例使用的是：

`public/assets/models/softmind/scene.gltf`

### 3.3 音频资源

- 一个条目可以有多条 `audioStems`
- 每条音轨都可以配置独立文件、默认开关、声部组、声像、音量

示例字段：

```json
{
  "id": "violin",
  "name": "小提琴",
  "file": "/assets/audio/The Sleeping Beauty Waltz/Violin_睡美人圆舞曲.mp3",
  "defaultEnabled": true,
  "group": "strings",
  "stereoPan": 0,
  "gain": 1
}
```

### 3.4 知识内容

- `knowledgeCards` 用于乐理知识、作曲家故事、校园叙事说明
- 当前侧边栏会展示全部卡片
- 当前 AR 场景只把前 3 张卡片映射成 3 个热点

如果后续你们要 6 个或更多热点，需要修改：

`src/components/ar/MindArScene.tsx`

里的 `hotspotPositions`

## 4. 怎么替换成正式素材

## 4.1 替换海报

把正式海报文件放到：

`public/assets/posters/`

然后修改条目里的：

```json
"posterImage": "/assets/posters/你的海报文件"
```

建议：

- 统一尺寸比例
- 保持识别图与海报逻辑分离
- 海报可偏视觉展示，识别图要保证追踪稳定

## 4.2 替换识别图和 `.mind`

把正式识别资源放到：

- `public/assets/markers/xxx.png`
- `public/assets/markers/xxx.mind`

然后修改条目里的：

```json
"targetImage": "/assets/markers/xxx.png",
"trackingTargetSrc": "/assets/markers/xxx.mind"
```

注意：

- `targetImage` 是展示给用户看的图片
- `trackingTargetSrc` 是 MindAR 的识别文件
- 两者应配套使用

当前项目里的默认识别资源，是通过脚本从 MindAR 示例复制过来的，占位用途而已：

`scripts/sync-demo-assets.mjs`

正式项目里你们可以：

- 直接手动把正式资源放到 `public/assets/markers/`
- 或者把 `scripts/sync-demo-assets.mjs` 改成同步你们自己的正式资源

## 4.3 替换 3D 模型

把正式模型放到：

`public/assets/models/`

然后修改条目里的：

```json
"modelUrl": "/assets/models/你的模型路径"
```

当前 AR 模型在场景中的缩放和位置由：

`src/components/ar/MindArScene.tsx`

控制，主要看这段参数：

- `position`
- `scale`
- `animation`

如果正式模型导入后太大、太小、悬空、穿模，就改这里。

## 4.4 替换分轨音频

把正式音频放到：

`public/assets/audio/`

然后修改对应条目的：

- `file`
- `defaultEnabled`
- `group`
- `stereoPan`
- `gain`

当前占位音频是脚本生成的：

`scripts/generate-demo-audio.mjs`

正式项目里可以不再运行这个脚本，直接换成真实录音文件。

## 5. 怎么新增一个正式乐器条目

最直接的方法：复制 `src/data/entries.json` 中现有一个条目，改成新条目。

一个新条目最少改这些字段：

```json
{
  "id": "cello-memory",
  "title": "大提琴示例",
  "subtitle": "用于说明该条目的副标题",
  "description": "用于说明该条目的简介",
  "orchestraZone": "弦乐组",
  "themeColor": "#7c3aed",
  "sceneType": "mindar-image",
  "targetImage": "/assets/markers/cello.png",
  "trackingTargetSrc": "/assets/markers/cello.mind",
  "posterImage": "/assets/posters/cello.svg",
  "modelUrl": "/assets/models/cello/scene.glb",
  "fallbackMode": "poster",
  "audioStems": [],
  "knowledgeCards": []
}
```

注意几点：

- `id` 必须唯一
- `id` 会直接出现在深链里
- 新增条目后，不需要改路由代码
- 首页、入口页、体验页会自动识别它
- 导出二维码/NFC 时也会自动带上它

## 6. NFC 具体怎么接

## 6.1 系统里哪里是 NFC 接口

严格来说，这个项目当前没有“浏览器内主动读 NFC”的代码。  
真正和 NFC 对接的接口是：

- `scripts/generate-links.mjs` 生成的 `nfcUrl`
- `src/lib/entries.ts` 生成的深链规则

也就是说，NFC 接入方式是：

1. 为每个条目生成一个 URL
2. 把 URL 写入 NFC 标签
3. 手机碰一碰后打开这个 URL

## 6.2 实际操作步骤

1. 先确定你的正式域名，例如：

```text
https://huaying.example.com
```

2. 导出正式链接：

```bash
npm run export:links -- --base-url https://huaying.example.com
```

3. 打开：

`generated/nfc-links.json`

4. 找到对应条目的：

- `nfcUrl`
- `qrPath`

5. 用手机 NFC 写卡工具，把 `nfcUrl` 写入实体标签

## 6.3 当前导出结果能做什么

- `generated/nfc-links.json`：给硬件或运营同学查每个条目的链接
- `generated/qrcodes/*.svg`：可直接交给设计或物料印刷

## 7. 建模接口具体预留在哪里

建模相关最核心的接口只有两个层级：

## 7.1 数据层接口

在：

`src/types/manifest.ts`

和：

`src/data/entries.json`

里，`modelUrl` 是模型入口。

你们后续无论是：

- 单乐器模型
- 乐手模型
- 小型室内乐
- 12 人舞台

都可以先统一接到 `modelUrl`。

## 7.2 场景层接口

在：

`src/components/ar/MindArScene.tsx`

里，模型的这些参数都已经预留：

- `src`
- `position`
- `scale`
- `animation`

如果后续你们想做得更复杂，可以继续往这里扩展：

- 多模型同时加载
- 乐手分区站位
- 模型点击反馈
- 更复杂的舞台灯光
- 多热点对应多个乐手

---

## 8. 乐理热点和交互接口怎么接

当前逻辑分两层：

### 8.1 数据层

在：

`src/data/entries.json`

里配置 `knowledgeCards`

### 8.2 展示层

- 侧边栏展示：`src/components/KnowledgePanel.tsx`
- AR 热点映射：`src/components/ar/MindArScene.tsx`

当前 AR 热点数量固定为 3 个。  
如果以后想让一个 12 人舞台每个乐手都能点，需要改 `MindArScene.tsx` 的热点布局逻辑，而不是改页面路由。

## 9. 现在这套代码更适合的工作分工

### 内容 / 美术同学

主要维护：

- `posterImage`
- `targetImage`
- `trackingTargetSrc`
- `knowledgeCards`

### 建模同学

主要维护：

- `modelUrl` 对应模型文件
- 模型尺寸、原点、贴图完整性

### 音频同学

主要维护：

- `audioStems`
- 分轨文件命名
- 音量和平衡参数

### 技术同学

主要维护：

- `entries.json` 条目装配
- `MindArScene.tsx` 场景布局
- `AudioEngine.ts` 音频交互逻辑
- `generate-links.mjs` 的正式导出

---

## 10. 正式上线前的建议检查表

- 每个条目都有唯一 `id`
- 每个条目的 `posterImage`、`targetImage`、`.mind`、`modelUrl` 都能访问
- 识别图印刷后仍能稳定触发追踪
- 安卓 NFC 标签能打开正确的正式域名
- iPhone 扫二维码能进入同一个条目
- 真实分轨音频长度一致，避免循环错位
- 正式模型在手机端不会过大导致卡顿
- 至少在一台安卓和一台 iPhone 上各跑一遍

---

## 11. 当前这套底座的边界

一期已经解决的是：

- 入口链路
- 内容配置化
- WebAR 骨架
- 分轨音频
- 热点知识卡

一期还没有做的是：

- 真正的浏览器内 Web NFC 读写
- 后台 CMS
- 正式 12 乐器全编制内容
- 多模型复杂舞台编排
- 用户数据统计与试点分析

如果二期要做，可以在当前结构上继续叠加，不需要推翻重来。
